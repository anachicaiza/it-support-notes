# Help Desk Workflow

## What Is A Ticket?

A ticket is a documented technical issue or service request used to track support tasks and troubleshooting progress.
Tickets help IT teams organize work and maintain documentation.


# Common Ticket Types

## Incident
An incident is an unexpected issue affecting normal operations.

Examples:
* No internet access
* Outlook not opening
* Printer failure

## Service Request
A service request is a user request for access, software or assistance.

Examples:
* Password reset
* Software installation
* Access request

# Ticket Priorities

## P1 — Critical

Major issue affecting many users or business operations.

Example:
* Company network outage

## P2 — High

Serious issue affecting important work.

Example:
* Manager unable to access system

## P3 — Medium

Normal issue affecting one user.

Example:
* Application problem

## P4 — Low

Minor inconvenience with low business impact.

Example:
* Desktop wallpaper issue


# Escalation

Escalation means transferring a ticket to a higher-level support team when additional expertise or permissions are required.


# Importance Of Documentation

Documentation is important because it:

* tracks troubleshooting steps
* improves consistency
* helps future troubleshooting
* supports teamwork

Good documentation is an important part of professional IT support.
